# Security Policy

## Supported Versions

Security fixes are provided for the latest release on the `main` branch.

## Reporting a Vulnerability

Please do not open public issues for security vulnerabilities.

Use one of the following:

- GitHub Security Advisories (preferred): create a private vulnerability report in the repository Security tab
- If Security Advisories are unavailable, contact maintainers through repository contact channels and request a private disclosure path

When reporting, include:

- A clear description of the vulnerability
- Reproduction steps or proof of concept
- Potential impact
- Suggested mitigation (if available)

## Response Expectations

- Initial triage response target: within 7 days
- Status updates: at least every 14 days until resolution

## Disclosure Process

- We validate and reproduce the report
- We prepare and test a fix
- We publish a release with notes and mitigation guidance
- We credit reporters unless they prefer to remain anonymous
